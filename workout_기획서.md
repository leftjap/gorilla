

## 운동 기록 웹앱 — 기획 정리

---

### 앱 개요

세트 쉬는 시간에 한 손으로 입력 완료할 수 있는 극도로 단순한 웨이트 트레이닝 기록 앱. PWA 웹앱으로 제작하고, 데이터는 LocalStorage + GAS 시트 동기화. 기존 이름표 프로젝트와 같은 기술 스택.

---

### 사용 흐름

체육관 도착 → 앱 열기 → **운동 시작** 탭 → 오늘 할 부위 태그 선택 (하체, 어깨, 데일리, 인터벌 — 탭한 순서대로 정렬) → 첫 태그 선택 시 타이머 자동 시작 → 종목 카드가 순서대로 펼쳐짐 → 세트별 무게/횟수 입력 + 체크 → 체크 시 휴식 타이머 자동 시작 + PR 즉시 판정 → 전종목 완료 → **운동 완료** 요약 카드 표시

---

### 화면 구조

#### 1. 홈 (대시보드)

앱 열면 바로 보이는 화면. 운동 시작 버튼이 가장 눈에 띄게.

**포함 요소:**
- **운동 시작 버튼** — 화면 최상단, 가장 크고 눈에 띄게
- **이번 주 요약 카드** — 운동 횟수, 총 볼륨(kg), 총 시간, 추정 칼로리
- **연속 기록 (스트릭)** — 연속 운동 일수
- **최근 PR** — 최근 갱신된 PR 1~2개 (종목명 + 이전→현재)
- **캘린더** — 월간 캘린더, 운동한 날에 부위별 컬러 도트 표시. 날짜 탭하면 그날 세션 요약 카드 펼침

#### 2. 운동 진행

앱의 핵심 화면. 세트 쉬는 시간(60~90초)에 입력이 끝나야 함.

**부위 태그 선택:**
- 화면 상단에 부위 태그 가로 나열 (가슴, 등, 하체, 어깨, 데일리, 인터벌)
- 탭하면 선택 + 선택 순서가 종목 표시 순서
- 선택 안 한 태그는 흐려지거나 사라짐
- 첫 태그 선택 시 운동 타이머 자동 시작

**종목 카드:**
- 선택한 부위의 종목이 미리 설정된 순서대로 아코디언 카드로 표시
- 현재 진행 중 종목이 펼쳐져 있고 나머지는 접힘
- 카드 상단에 **오늘 볼륨 vs 지난번 볼륨** 실시간 비교

**세트 테이블 (종목 카드 안):**
- 각 행: 세트 번호 | 무게(kg) | 횟수 | 워밍업(W) 토글 | 체크 버튼
- 지난번 동일 종목·동일 세트의 값이 연하게 표시
- **체크만 탭하면 지난번 값으로 자동 채움** (1탭 완료)
- 값을 바꿀 때만 숫자 수정 후 체크 (2~3탭)
- 워밍업(W) 토글: 해당 세트를 PR/볼륨 집계에서 제외, 시각적으로 연하게
- 세트 추가 버튼

**PR 즉시 감지:**
- 세트 체크 순간에 PR 여부 판정
- PR이면 해당 세트 행에 금색 플래시 + "🏆 개인 기록!" 인라인 표시

**휴식 타이머:**
- 세트 체크 시 자동 카운트다운 시작 (기본 90초)
- `Date.now()` 기반으로 종료 시각 저장 — 화면 꺼졌다 켜져도 남은 시간 정확히 표시
- 소리/진동 없음, 시각 표시만 (큰 숫자 카운트다운)
- 0이 되면 색상 변화로 "쉬는 시간 끝" 표시
- 종목별 기본 휴식 시간 설정 가능 (데이터에 defaultRestSec)

**운동 완료 요약 카드:**
- 운동 완료 버튼 탭 시 표시
- 총 운동 시간
- 총 볼륨 (워밍업 제외) + 지난 동일 부위 조합 대비 ±
- 추정 칼로리 소모
- PR 갱신 카드 (종목별, 이전→현재)
- 종목 수 / 총 세트 수
- 히스토리에서 다시 열어볼 수 있음

#### 3. 기록/통계

**기간별 통계 카드:**
- 주간/월간 토글
- 볼륨(kg), 칼로리(kcal), 시간(분) — 숫자 + 미니 추이 차트
- 이전 기간 대비 증감 표시

**캘린더 뷰:**
- 홈 캘린더의 넓은 버전
- 날짜 탭 → 세션 상세

**인바디 기록:**
- 체중(kg), 체지방률(%), 골격근량(kg) 수동 입력
- 측정 날짜와 함께 저장
- 시계열 라인 그래프 (점이 찍힌 만큼만 표시)
- 최신 체중은 칼로리 추정에 자동 참조
- 1차에는 핵심 3개 항목만, 나중에 필요하면 필드 추가

**세션 히스토리 리스트:**
- 날짜, 부위 태그, 볼륨, 시간으로 구성된 카드 리스트
- 탭하면 해당 세션의 완료 요약 카드 표시

#### 4. 설정

- 1차에는 UI 없이 코드 하드코딩
- 2차에 UI 제공: 종목 편집, 부위별 종목 순서, 기본 세트/횟수, 기본 휴식 시간, 체중 입력

---

### 부위 그룹 & 종목 (하드코딩 초기값)

**하체:** 레그 익스텐션, 레그 컬, 바벨 백 스쿼트, 레그 프레스

**어깨:** 오버헤드 프레스, 사이드 레터럴 레이즈

**가슴 (상체A 계열):** 인클라인 벤치프레스

**등 (상체A 계열):** 랫풀다운, 시티드 로우, 페이스 풀, 바벨 로우, 데드리프트

**데일리:** 바벨 리스트 컬, 싯업

**인터벌:** 러닝 (시간만 입력, 무게/횟수 없음)

---

### 데이터 스키마

**종목 마스터**
```
{ id, name, bodyPart, equipment, 
  defaultSets, defaultReps, defaultRestSec, 
  met, sortOrder }
```

**세션 로그**
```
{ id, date, startTime, endTime,
  tags: ['lower', 'shoulder', 'daily', 'interval'],
  exercises: [
    { exerciseId, sortOrder, sets: [
      { weight, reps, done, isWarmup, isPR, restSec }
    ]}
  ],
  totalVolume, totalVolumeExWarmup,
  totalCalories, durationMin, memo }
```

**PR 기록**
```
{ exerciseId, records: [
  { weight, reps, volume, estimated1RM, date, sessionId }
]}
```

**인바디 기록**
```
{ id, date, weight, bodyFatPct, muscleMass, memo }
```

### 칼로리 추정 방식

`칼로리(kcal) = MET × 체중(kg) × 시간(시간) × 1.05`

- 웨이트: MET 3~6 (종목별 기본값)
- 유산소 인터벌: MET 8~10
- 체중: 인바디 기록의 최신값 또는 설정 입력값
- 정밀도가 목적이 아님, 피드백용 추정치

---

### 기술 구조

- 단일 HTML + JS + CSS (기존 이름표 프로젝트와 동일 패턴)
- 데이터: LocalStorage 저장 + GAS 시트 동기화
- PWA: manifest + service worker (홈 화면 추가, 기본 오프라인)
- GAS: 기존 백엔드에 시트 추가

---

### 웹앱 제약사항

- **진동 불가** — iOS Safari가 Vibration API 미지원. 알림은 시각 표시만.
- **백그라운드 제한** — 화면 꺼지거나 다른 앱 전환 시 `setInterval` 멈춤. `Date.now()` 기반으로 복귀 시 남은 시간 재계산하여 해결.
- **소리 제한** — iOS Safari에서 사용자 인터랙션 없이 자동 재생 불가. 1차에서는 소리 알림 미포함.

### 1차 MVP vs 2차

| 1차 (바로 만들 것) | 2차 (나중에) |
|---|---|
| 부위 태그 선택 → 종목 카드 → 세트 입력 | 종목/루틴 편집 UI (설정 화면) |
| 지난번 값 프리필 + 1탭 체크 자동 채움 | 종목 대체 (스쿼트→레그프레스 전환) |
| 워밍업 세트 태그 (W 토글) | 종목 순서 드래그 변경 |
| PR 즉시 감지 + 인라인 표시 | 종목별 진행 그래프 |
| 휴식 타이머 (시각만, Date.now 기반) | 운동 사진 첨부 |
| 운동 완료 요약 카드 | 공유용 요약 이미지 생성 |
| 캘린더 (부위별 컬러 도트) | PWA 오프라인 캐시 강화 |
| 주간/월간 통계 (볼륨/칼로리/시간) | 소리 알림 (오디오 unlock 방식) |
| 인바디 수동 입력 + 추이 그래프 | |
| 세션 히스토리 리스트 | |
| LocalStorage + GAS 시트 동기화 | |
